# GitHub Repo Setup Checklist

Apply these settings in the GitHub web UI after pushing v1.0.

---

## Settings → General → About (top-right of repo page, click the gear)

| Field | Value |
|---|---|
| **Description** | Sanitized demo of a production PHP admin panel — payments, subscriptions, e-learning, webinars, email campaigns |
| **Website** | `https://demo-panel.infinityfreeapp.com` |
| **Topics** | `php` `mysql` `admin-dashboard` `tailwindcss` `portfolio` `demo` `elearning` `email-campaigns` |

Tick **☑ Releases**, **☑ Packages** unchecked, **☑ Deployments** unchecked.

---

## Settings → General → Features

| Feature | State |
|---|---|
| Wikis | Off — README covers everything |
| Issues | **On** — looks more like a maintained project |
| Discussions | Off |
| Projects | Off |

---

## Profile — pin this repo

1. Go to your GitHub profile: **github.com/SyedIhsan**
2. Click **Customize your pins** (top-right of the pinned repos section)
3. Tick **admin-panel** → Save

---

## Social preview image (optional but recommended)

A 1280×640 social card shown when the repo URL is shared on LinkedIn/Twitter.

1. Settings → General → Social preview → **Edit**
2. Upload `screenshots/02-dashboard.png` cropped to 1280×640
   (or any screenshot — the dashboard is the most impressive at a glance)
